一、实现图片懒加载的方法
1、为什么要实现图片懒加载？

因为屏幕的视口大小是有限的，有的时候可能会出现加载了100张图片，但是实际页面只能展示50张图片，这样的情况下，就会浪费加载的资源，所以需要使用懒加载，实现看到多少就加载多少的效果，也可以减轻服务器的压力。尤其是对于移动端来说，往往一次只能看到少数照片，不需要一次性加载所有的图片。
2、实现原理
当图片进入视口访问范围内时再发送请求获取对应的图片。
3、具体方法
（1）使用IntersectionObserver接口
https://q.qq.com/wiki/develop/miniprogram/API/QML/IntersectionObserver.html
IntersectionObserver接口可以观察目标元素与祖先元素交叉的状态，即判断内容是否在视口内。具体代码如下：
// 创建观察对象
const observer = new IntersectionObserver((entries)=>{
entries.forEach(entry => {
if(entry.isIntersecting){
// 元素在视口范围内
const target = entry.target
target.src = entry.target.getAttribute(“data-src”)
target.removeAttribute(“data-src”)
// 取消观察已加载元素
observer.unobserve(target)
}
})
})
// 指定被观察元素
const imgDOMs= document.getElementsByTagName(“img”)
Array.from(imgDOMs).forEach(item => {
observer.observe(item)
})
（1）使用滚动监听+scrollTop+offsetTop+innerHeight
原理：当视口高度window.innerHeight+内容偏移量window.pageYOffset > 图片距离顶部的高度offsetTop时，显示图片。具体实现

二、手写new
主要步骤
创建空对象，空对象的原型指向构造函数的原型对象，构造函数的this指向空对象，执行返回的结果。
// 只是实现new关键字，并不是写了个构造方法呀，八嘎
function mynew(Func,…args){
// 1、创建一个新对象
const obj = {}
// 2、新对象的原型指向构造函数的原型对象
obj.proto = Func.prototype
// 3、将构造函数的this指向新对象
const result = Func.apply(obj,args)
// 4、返回执行结果
return result instanceof Object ? result : obj
}

function Person(name,age){
this.name = name
this.age = age
}
Person.prototype.sayHello = function (){
console.log(Hello, my name is ${this.name} and I am ${this.age} years old.);
}

// 错误写法，箭头函数的this指向有误
Person.prototype.sayHello = () =>{
console.log(Hello, my name is ${this.name} and I am ${this.age} years old.);
}

const person = mynew(Person,”tan”,20)
person.sayHello()
